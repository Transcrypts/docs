---

## Overview

Creates an assets report for a user within the Transcrypts system.

## Authentication

This endpoint requires credentials via header authentication:
- **ClientID** (Header)
- **AccessKey** (Header)

## Path Parameters

<ParamField path="user_id" type="string" required>
  The unique identifier for the user.
</ParamField>

## Response

### Success Response (201)

The endpoint returns a confirmation upon successful creation of the assets report. The response structure contains the report details for the newly created asset verification report.

### Error Responses

| Status Code | Description |
|-------------|-------------|
| **400** | Bad Request - Request parameters are incorrect or missing required fields |
| **401** | Unauthorized - Authentication credentials are invalid or missing |
| **403** | Forbidden - Request lacks proper authentication credentials |

<Note>
  The endpoint supports assets verification as a product type within the Bridge SDK workflow. Users can initialize Bridge with `product_type: "assets"` to guide users through the asset verification process before calling this endpoint.
</Note>
