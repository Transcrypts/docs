## Overview

Direct Deposit Switch (DDS) reports contain attributes for managing direct deposit configurations. This documentation covers the report structure, attributes, and available endpoints.

## Attributes

The Deposit Switch Report includes the following attributes:

| Attribute | Type | Description |
|-----------|------|-------------|
| id | string | Unique identifier |
| status | string | Status from Connection Lifecycle |
| finished_at | string | Time when report was finished |
| completed_at | string | Time when report was completed (Deprecated, invalid datetime format) |
| access_token | string | Access token for Link to payroll provider |
| tracking_info | string | Information passed to Transcrypts Bridge from partner |
| deposit_details | object | Bank account info for DDS and PLL (see Deposit details object) |
| initial_accounts | object | List of initial accounts (see Initial accounts object) |

## Deposit Details Object

This object contains banking information for deposit configuration:

| Attribute | Type | Description |
|-----------|------|-------------|
| account_number | string | Account number |
| account_type | string | Account type: `checking` or `savings` |
| routing_number | string | Routing number |
| bank_name | string | Bank name |
| deposit_type | string | `entire`, `percent`, or `amount` |
| deposit_value | string | Deposit value |

## Initial Accounts Object

Information about the initial accounts:

| Attribute | Type | Description |
|-----------|------|-------------|
| account_number | string | Account number |
| routing_number | string | Routing number |

## Endpoints

Two endpoints retrieve deposit switch report information:

- Retrieve a direct deposit switch report for a link
- Retrieve a paycheck linked loan report
