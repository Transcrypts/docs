## Overview

This page provides comprehensive reference information for insurance data responses within the Transcrypts API system.

## Main Attributes

The top-level insurance report response includes:

| Attribute | Type | Description |
|-----------|------|-------------|
| id | string | Unique identifier |
| status | string | Connection lifecycle status |
| finished_at | string | Report completion timestamp |
| access_token | string | Link access token for payroll provider |
| tracking_info | string | Partner-passed information |
| insurance | object | Insurance details collection |
| provider | string | Insurance provider identifier |
| identity | object | Person's identity information |
| pdf_report | uri | PDF format report |

## Insurance Object Structure

The insurance object contains five main collections:

- **policies**: List of policy records
- **drivers**: List of driver information
- **addresses**: List of address records
- **driving_records**: List of driving history
- **agents**: List of agent information

### Policies Object

Core policy attributes include policy type, dates (effective, expiry, renewal, canceled), premium amounts, carrier information, and status flags.

**Policy Types Supported**: `AUTO`, `BOAT`, `CONDO`, `FARMOWNER`, `FLOOD`, `HOMEOWNERS`, `LANDLORD`, `LIFE`, `MOTORCYCLE`, `RENTERS`, `UMBRELLA`, `COMMERCIAL_AUTO`, `CYBER`, `IDENTITY_THEFT`, and others.

**Policy Status Values**: `ACTIVE`, `CANCELLED`, `EXPIRED`, `UNVERIFIED`, `PENDING_ACTIVATION`, `PENDING_CANCELLATION`

#### Dwellings Object

Residential properties under homeowners policies include:

- Replacement and cash value amounts
- Coverage items with premiums and limits
- Mortgagee information (first, second, third positions)

**Coverage Names**: `DWELLING`, `OTHER_STRUCTURE`, `PERSONAL_PROPERTY`, `LOSS_OF_USE`, `PERSONAL_LIABILITY`, `WINDSTORM_OR_HAIL`, `EARTHQUAKE`, `JEWELRY`, `MUSICAL_INSTRUMENT`, and similar specialized coverages.

#### Vehicles Object

Vehicle records contain:

- Year, make, model, series information
- VIN and annual mileage
- Garaging address and lien holder details
- Associated coverage items

**Vehicle Coverage Types**: `LIABILITY` (bodily injury, property damage), `COMPREHENSIVE`, `COLLISION`, `UNINSURED_MOTORIST` variants, `TOWING_AND_LABOR`, `RENTAL_REIMBURSEMENT`, and others.

### Drivers Object

Driver records include personal identification:

| Attribute | Description |
|-----------|-------------|
| first_name, middle_name, last_name | Name components |
| drivers_license_number | Driver's license number |
| drivers_license_state | License issuing state |
| date_of_birth | Date of birth |
| gender | `MALE`, `FEMALE`, `NONBINARY` |
| marital_status | `SINGLE`, `MARRIED`, `DIVORCED`, `WIDOWED`, `SEPARATED`, `CIVIL_UNION` |
| relationship_to_insured | `INSURED`, `SPOUSE`, `PARENT`, `CHILD`, `SIBLING` |

### Named Insureds Object

Personal policy holders contain:

- Full name components
- Primary insured flag
- Simple identifier

### Commercial Named Insureds Object

Business entity records include:

| Attribute | Description |
|-----------|-------------|
| business_name | Business name |
| form_of_business | `INDIVIDUAL`, `PARTNERSHIP`, `LLC`, `S_CORPORATION`, `CORPORATION`, `TRUST`, etc. |
| gl_code | GL classification code |
| sic_naics_code | SIC/NAICS classification code |
| fein | Federal Employer Identification Number |
| ssn | Social Security Number |
| is_primary | Primary status flag |
| address | Business address |

### Addresses Object

All addresses contain:

| Attribute | Description |
|-----------|-------------|
| street | Street address |
| city | City |
| state | State |
| zip | ZIP code |
| country | Country designation |
| id | Unique identifier |

### Driving Records Object

Historical driving incidents show:

| Attribute | Description |
|-----------|-------------|
| incident_date | Date of incident |
| type | `ACCIDENT` or `VIOLATION` |
| violation | Violation specifics when applicable |

**Violation Categories**: `SPEEDING`, `RECKLESS_DRIVING`, `DUI`, `HIT_AND_RUN`, `DISTRACTED_DRIVING`, `FAILURE_TO_YIELD`, etc.

### Claims Object

Insurance claim records include:

| Attribute | Description |
|-----------|-------------|
| claim_id | Claim identifier |
| carrier_reference | Carrier reference number |
| occurrence_date | Date of occurrence |
| closure_date | Date claim was closed |
| claim_type | Type of claim |
| status | `OPEN` or `CLOSED` |
| payout_amount | Payout amount in cents |
| representative | Contact information |
| address | Associated address |

**Claim Types**: `WIND_AND_HAIL_DAMAGE`, `WATER_DAMAGE_AND_FREEZING`, `FIRE_AND_LIGHTNING_DAMAGE`, `PERSONAL_LIABILITY`, `THEFT`, `CAR_ACCIDENT`, `VANDALISM`, `HURRICANE_DAMAGE`, etc.

### Loss Events Object

Separate from claims, loss events track:

| Attribute | Description |
|-----------|-------------|
| occurrence_date | Date of occurrence |
| claim_date | Date claim was filed |
| event_type | Event type classification |
| amount_paid | Amount paid |
| amount_reserved | Amount reserved |
| subrogation | Subrogation flag |
| is_open | Open status flag |

**Loss Event Types**: `BURGLARY_AND_THEFT`, `WATER_DAMAGE`, `WIND_AND_HAIL`, `FIRE`, `CUSTOMER_SLIPS_AND_FALLS`, `PRODUCT_LIABILITY`, `VEHICULAR_ACCIDENT`, `WINDSHIELD_DAMAGE`, etc.

### Documents Object

Supporting policy documents include:

| Attribute | Description |
|-----------|-------------|
| title | Document title |
| document_id | Document ID |
| type | Type classification |
| date_added | Date added (effective date) |
| file_url | File URL reference |

**Document Types**: `INSURANCE_APPLICATION`, `INSURANCE_BINDER`, `DECLARATIONS`, `ENDORSEMENT`, `INSURANCE_ID_CARD`, `VERIFICATION_OF_INSURANCE`

## Identity Object Reference

The identity section contains personal identity verification information relevant to the insurance account.

<Note>All monetary amounts in the API are represented in cents (integer values). Date-time fields follow standard formats. Boolean fields indicate status flags for various policy conditions and qualifications.</Note>
