# Get Uploaded File Details

This endpoint retrieves detailed information about a specific uploaded file within a document collection, including its processing status and any recognized documents.

## Authentication

This endpoint requires API credentials:
- **ClientID** (header)
- **AccessKey** (header)

## Path Parameters

| Parameter | Type | Description |
|-----------|------|-------------|
| `collection_id` | string | The unique identifier for the document collection (required) |
| `uploaded_file_id` | string | The unique identifier for the uploaded file (required) |

## Response (200 OK)

```json
{
  "file_id": "f1234567890abcdef1234567890abcde",
  "filename": "merged_paystubs.pdf",
  "mime_type": "application/pdf",
  "status": "successful",
  "file_url": "https://example.com/files/file123xyz789.pdf",
  "validations": {
    "is_viable_size": true,
    "is_supported_type": true,
    "is_accessible": true,
    "is_valid": true,
    "is_readable": true,
    "is_unique": true
  },
  "user_id": "a1b2c3d4e5f6478899aabbccddeeff00",
  "external_user_id": "ext_user_789",
  "created_at": "2025-11-11T10:00:00Z",
  "updated_at": "2025-11-11T10:05:00Z"
}
```

### Response Fields

| Field | Type | Description |
|-------|------|-------------|
| `file_id` | string | Unique identifier for the file |
| `filename` | string | Name of the uploaded file |
| `mime_type` | string | File type (e.g., `application/pdf`, `image/jpeg`) |
| `status` | string | Processing status: `pending`, `validating`, `validated`, `invalid`, `duplicate`, `processing`, `successful`, or `failed` |
| `file_url` | string (URI) | Direct URL to access the file |
| `validations` | object | File validation results |
| `user_id` | string (nullable) | Associated Transcrypts user ID if assigned |
| `external_user_id` | string (nullable) | Associated external system user ID if assigned |
| `created_at` | string (date-time) | File creation timestamp |
| `updated_at` | string (date-time) | Last update timestamp |

### Validations Object

| Field | Type | Description |
|-------|------|-------------|
| `is_viable_size` | boolean | File size is within acceptable range |
| `is_supported_type` | boolean | File type is supported |
| `is_accessible` | boolean | File can be accessed |
| `is_valid` | boolean | File passes validation |
| `is_readable` | boolean | File content is readable |
| `is_unique` | boolean | File is not a duplicate |
