# Document Collections

Document Collections provide a mechanism to upload, process, and manage multiple documents in a single workflow. Documents are automatically validated and processed using AI-powered categorization to identify document types such as paystubs, W2s, 1099s, and more.

## Quickstart

### Python Setup

Save this as `prepare_request.py`:

```python
import base64, json, sys

with open(sys.argv[1], 'rb') as f:
    content = base64.b64encode(f.read()).decode()

json.dump({
    "documents": [{
        "mime_type": "application/pdf",
        "content": content,
        "filename": sys.argv[1]
    }]
}, open('request.json', 'w'))
```

### Execution

```bash
python prepare_request.py mydocument.pdf
curl -X POST https://prod.transcrypts.com/v1/documents/collections/ \
    -H "Content-Type: application/json" \
    -d @request.json
```

## Collection Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `collection_id` | string | Unique identifier for the collection |
| `created_at` | date-time | Creation timestamp |
| `updated_at` | date-time | Last update timestamp |
| `files_count` | integer | Number of uploaded files |
| `documents_count` | integer | Number of recognized documents |
| `users_count` | integer | Associated user count |

## Uploaded File Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `file_id` | string | Unique file identifier |
| `filename` | string | Name of uploaded file |
| `mime_type` | string | File MIME type (pdf, jpeg, png, tiff, webp, bmp, heic, heif) |
| `status` | string | Processing status: `pending`, `validating`, `validated`, `invalid`, `duplicate`, `processing`, `successful`, `failed` |
| `validations` | object/null | Validation results (`is_viable_size`, `is_supported_type`, `is_accessible`, `is_valid`, `is_readable`, `is_unique`) |
| `user_id` | string/null | Associated Transcrypts user ID |
| `external_user_id` | string/null | External system user ID |

## Recognized Document Attributes

| Attribute | Type | Description |
|-----------|------|-------------|
| `document_id` | string | Document identifier |
| `file_id` | string | Parent file ID |
| `document_type` | string | Type classification (see Document Types below) |
| `document_subtype` | string/null | Subtype (see Document Subtypes below) |
| `status` | string | Processing status: `successful`, `failed`, `rejected` |
| `first_name` | string/null | Extracted first name |
| `last_name` | string/null | Extracted last name |
| `user_id` | string/null | Associated user ID |
| `external_user_id` | string/null | External user ID |
| `start_page` | integer | Starting page number |
| `end_page` | integer | Ending page number |

## Finalization Process

After documents are processed and categorized, use the finalize endpoint to create links for the recognized documents. This converts pre-processed documents into usable Transcrypts links for retrieving income and employment data.

### User Object

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | string | Transcrypts user ID |
| `external_user_id` | string/null | External system user ID |
| `links` | array | Associated links created/updated |

### Link Object

| Attribute | Type | Description |
|-----------|------|-------------|
| `link_id` | string | Transcrypts link ID |
| `status` | string | Link status (`new`, `parse`, `full_parse`, `done`, `no_data`, `config_error`, `error`, `unavailable`) |
| `documents` | array | Associated documents |

### Document Object (Finalization Response)

| Attribute | Type | Description |
|-----------|------|-------------|
| `id` | string | Document ID |
| `document_type` | string | Document type (uppercase) |
| `document_subtype` | string/null | Document subtype (uppercase, optional) |
