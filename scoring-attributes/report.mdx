## Overview

This page documents the Scoring Attributes Report API, which provides financial analysis and risk assessment through various metrics and derived income calculations.

## Main Attributes

The report response includes these key fields:

| Attribute | Type | Description |
|-----------|------|-------------|
| report_id | string | Unique identifier generated for the report |
| source_guid | string | GUID of prior report for refresh reports; null for new customers |
| refresh | Boolean | Status indicating if this is a refresh report |
| lender_reference | string | Value to identify report (e.g., loan application number) |
| customer_identifier | string | Customer-level unique identifier (avoid PII like SSN) |
| purpose | string | Report purpose: `decisioning`, `verification`, or `analytics` |
| status | string | Terminal statuses: `success`, `failed`, `data_import_error`, or `processing` |
| cutoff_date | string | Latest date of financial activity included |
| created_at | string | Date and time of report request |
| updated_at | string | Date and time of report update |
| alerts | array | Notification objects for specific conditions |
| scores | array | Generic or custom scoring models |
| metrics | array | Financial metrics with time periods |
| derived_incomes | array | Recurring income streams identified |

## Alerts Object

Alerts notify lenders of conditions that don't prevent report completion:

| Alert Code | Message |
|-----------|---------|
| 1 | No consistent income sources present |
| 2 | No consistent payment streams present |
| 3 | Short transaction history |
| 4 | Low transaction activity |
| 5 | Unable to calculate daily balances for all accounts |
| 6 | Accounts with stale ending balances are present |
| 7 | Low ratio of debit activity |

## Scores Object

Contains scoring model information:

| Attribute | Type | Description |
|-----------|------|-------------|
| name | string | Model name |
| version | float | Model version number |
| value | string | Scaled score value |

## Metrics Object

Financial metrics with configurable time periods:

| Attribute | Type | Description |
|-----------|------|-------------|
| name | string | Metric name (excluding time period) |
| time_period | string | Period abbreviation (e.g., `1m_l0` for last month, `3m_l0` for last 3 months, `lftm` for lifetime) |
| short_name | string | Abbreviated metric name with period |
| unit | string | Measurement type: `dollars`, `percent`, `boolean`, or `count` |
| value | number | Numeric value; null values must be parsed separately (not equivalent to 0) |

**Value Formatting:**
- Amounts: Dollar values with decimal cents (e.g., 2.99)
- Percentages: Integer representation (5% = 5)
- Booleans: 1 for True, 0 for False
- Counts: Integer values

## Derived Incomes Object

Identified recurring income streams:

| Attribute | Type | Description |
|-----------|------|-------------|
| description | string | Normalized income source description (sensitive info redacted) |
| frequency | string | Recurrence pattern: `monthly`, `semi_monthly`, `bi_weekly`, `weekly` |
| days | array | Day(s) of recurrence; -1 indicates end of month |
| formatted_frequency | string | Human-readable frequency with days |
| months | number | Longest recurring period in whole months (2, 3, 4, 6 month intervals evaluated) |
| average_amount | double | Average deposit amount in covered interval |
| monthly_amount | double | Projected monthly amount based on frequency and average |

## Available Endpoints

Two endpoints support scoring attributes reporting:

- **POST** - Create a scoring attributes report
- **GET** - Retrieve a scoring attributes report
